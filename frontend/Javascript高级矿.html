JavaScript 中那些奇葩的点 ​

JavaScript 作为一门流行的编程语言，以其灵活性和动态性广受欢迎，但也因其不按常理出牌的行为让开发者头疼不已。本文总结了一些 JavaScript 中的奇葩点。

1. 类型转换 ​

1.1 自动类型转换 ​

JavaScript 会在需要时自动进行类型转换，有时会导致意外结果。

js
console.log('5' - 1); // 4
console.log('5' + 1); // '51'
console.log(true + false); // 1
console.log(1 + '2' + '2'); // '122'
console.log(1 + +'2' + '2'); // '32'
console.log(1 + -'1' + '2'); // '02'

1.2 特殊的相等比较 ​

使用 == 比较时会进行类型转换。

js
console.log(null == undefined); // true
console.log('' == 0); // true
console.log(false == '0'); // true
console.log([] == []); // false
console.log({} == {}); // false

2. 数组和对象 ​

2.1 数组和对象的相等性 ​

空数组和空对象的比较结果出乎意料。

js
console.log([] == []); // false
console.log({} == {}); // false
console.log([] == ![]); // true

js

5.1 NaN 的奇怪行为 ​

NaN 不等于任何值，包括它自身。

js
console.log(NaN === NaN); // false
console.log(typeof NaN); // 'number'
console.log(isNaN(NaN)); // true

总结 ​
