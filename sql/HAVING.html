HAVING ​

HAVING 是SQL中用于过滤分组结果的关键字，它类似于 WHERE，但专门用于 GROUP BY 之后。

HAVING vs WHERE 的区别 ​

  • WHERE：在分组之前过滤行
  • HAVING：在分组之后过滤分组

基本语法 ​

sql
SELECT column1, aggregate_function(column2)
FROM table
WHERE condition
GROUP BY column1
HAVING condition
ORDER BY column1;

实际示例 ​

sql
-- 查找年龄分组中人数大于1的组
SELECT age, COUNT(*) as count
FROM users 
GROUP BY age 
HAVING COUNT(*) > 1;

-- 查找平均年龄大于25的部门
SELECT department, AVG(age) as avg_age
FROM employees 
GROUP BY department 
HAVING AVG(age) > 25;

-- 查找总销售额超过10000的客户
SELECT customer_id, SUM(amount) as total_amount
FROM orders 
GROUP BY customer_id 
HAVING SUM(amount) > 10000;

为什么需要HAVING？ ​

sql
-- 错误示例：WHERE不能用于聚合函数
SELECT age, COUNT(*) as count
FROM users 
GROUP BY age 
WHERE COUNT(*) > 1;  -- ❌ 错误！

-- 正确示例：使用HAVING
SELECT age, COUNT(*) as count
FROM users 
GROUP BY age 
HAVING COUNT(*) > 1;  -- ✅ 正确！

复杂示例 ​

sql
-- 查找2023年订单数量超过5个且总金额超过1000的客户
SELECT 
    customer_id,
    COUNT(*) as order_count,
    SUM(amount) as total_amount
FROM orders 
WHERE order_date >= '2023-01-01' 
    AND order_date < '2024-01-01'
GROUP BY customer_id 
HAVING COUNT(*) > 5 AND SUM(amount) > 1000
ORDER BY total_amount DESC;

记忆技巧 ​

  • WHERE：过滤行（Row）
  • HAVING：过滤组（Group）
  • HAVING 只能用于 GROUP BY 之后
  • HAVING 可以使用聚合函数（COUNT、SUM、AVG等）

简单来说，HAVING 就是分组后的 WHERE！
