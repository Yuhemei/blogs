概览 ​

SQL 的“增删改查”对应：

  • 增：INSERT
  • 删：DELETE
  • 改：UPDATE
  • 查：SELECT

下面给出常用语法与简洁示例（通用 SQL，附带少量 PostgreSQL/ MySQL 差异提示）。

INSERT（增） ​

  • 基本语法：
sql
INSERT INTO table_name (col1, col2, ...)
VALUES (val1, val2, ...);
1
2
  • 多行插入：
sql
INSERT INTO users (name, email)
VALUES ('Tom', 't@a.com'),
       ('Jerry', 'j@a.com');

  • 从查询插入：
sql
INSERT INTO archive_users (id, name)
SELECT id, name FROM users WHERE is_deleted = true;

  • 返回插入结果（PostgreSQL）：
sql
INSERT INTO users (name, email)
VALUES ('Anna', 'a@a.com')
RETURNING id, created_at;

  • UPSERT（冲突更新，PostgreSQL）：
sql
INSERT INTO users (email, name)
VALUES ('a@a.com', 'Anna')
ON CONFLICT (email)
DO UPDATE SET name = EXCLUDED.name, updated_at = NOW();

DELETE（删） ​

  • 基本语法（务必加条件）：
sql
DELETE FROM table_name
WHERE condition;
1
2
  • 示例：
sql
DELETE FROM users
WHERE last_login_at < NOW() - INTERVAL '1 year';

  • 级联删除（依赖外键设置 ON DELETE CASCADE），谨慎使用。

SELECT（查） ​

  • 基本语法：
sql
SELECT col1, col2
FROM table_name
WHERE condition
GROUP BY col1
HAVING condition_on_group
ORDER BY col1 DESC, col2 ASC
LIMIT n OFFSET m;

事务与并发（重要） ​

  • 事务包裹多步写操作，确保原子性：
sql
BEGIN;
  UPDATE accounts SET balance = balance - 100 WHERE id = 1;
  UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT; -- 或 ROLLBACK;
1
2
3
4
  • 隔离级别/锁与并发一致性，视业务选择（READ COMMITTED/REPEATABLE READ/SERIALIZABLE）。

性能与安全要点 ​

  • 使用参数化查询，避免 SQL 注入（例如 WHERE id = ?）。
  • 为过滤、连接、排序、分组字段建立合适索引。
  • 写操作加 WHERE，先 SELECT 验证，再执行写。
  • 批量写入使用多值 INSERT 或批处理；大事务拆分。
  • 需要返回新/改行时用 PostgreSQL RETURNING，可减少往返。
