SQL基础 ​

SQL（Structured Query Language，结构化查询语言）是用于管理关系型数据库的标准语言。无论是数据分析师、后端开发者还是数据科学家，掌握SQL都是必备技能。

什么是SQL ​

SQL是一种声明式语言，你只需要描述你想要什么数据，而不需要告诉数据库如何获取这些数据。SQL主要用于：

  • 数据查询：从数据库中检索数据
  • 数据操作：插入、更新、删除数据
  • 数据定义：创建、修改、删除数据库结构
  • 数据控制：管理用户权限和访问控制

SQL语法基础 ​

基本语法规则 ​

  1. SQL语句以分号结尾
  2. 关键字不区分大小写（但建议使用大写）
  3. 字符串用单引号包围
  4. 注释使用 -- 或 /* */
sql
-- 单行注释
/* 
   多行注释
*/

SELECT * FROM users;  -- 查询所有用户

数据类型 ​

数值类型 ​

  • INT / INTEGER：整数
  • DECIMAL(p,s)：精确小数
  • FLOAT：浮点数
  • DOUBLE：双精度浮点数

字符串类型 ​

  • VARCHAR(n)：可变长度字符串
  • CHAR(n)：固定长度字符串
  • TEXT：长文本

日期时间类型 ​

  • DATE：日期（YYYY-MM-DD）
  • TIME：时间（HH:MM:SS）
  • DATETIME：日期时间
  • TIMESTAMP：时间戳

其他类型 ​

  • BOOLEAN：布尔值
  • BLOB：二进制数据

子查询 ​

sql
-- 标量子查询
SELECT name FROM users WHERE age = (SELECT MAX(age) FROM users);

-- 列子查询
SELECT name FROM users WHERE age IN (SELECT age FROM users WHERE name LIKE '张%');

-- 行子查询
SELECT * FROM users WHERE (age, name) = (SELECT MAX(age), name FROM users WHERE name LIKE '张%');

-- EXISTS子查询
SELECT name FROM users u WHERE EXISTS (
    SELECT 1 FROM posts p WHERE p.user_id = u.id
);

性能优化技巧 ​

  1. 使用适当的索引
  2. **避免SELECT ***
  3. 使用LIMIT限制结果
  4. 合理使用WHERE条件
  5. 避免在WHERE子句中使用函数
sql
-- 不好的写法
SELECT * FROM users WHERE YEAR(created_at) = 2023;

-- 好的写法
SELECT id, name, email FROM users 
WHERE created_at >= '2023-01-01' AND created_at < '2024-01-01';

最佳实践 ​

1. 命名规范 ​

  • 表名使用复数形式：users, orders
  • 列名使用下划线分隔：user_id, created_at
  • 避免使用SQL关键字作为列名

2. 数据类型选择 ​

  • 选择合适的数据类型和长度
  • 使用NOT NULL约束提高性能
  • 合理使用AUTO_INCREMENT

3. 查询优化 ​

  • 使用EXPLAIN分析查询计划
  • 避免复杂的子查询
  • 合理使用索引

4. 安全考虑 ​

  • 使用参数化查询防止SQL注入
  • 限制用户权限
  • 定期备份数据

5. 代码示例 ​

sql
-- 创建用户表的最佳实践
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    birth_date DATE,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_email (email),
    INDEX idx_username (username),
    INDEX idx_active_created (is_active, created_at)
);

-- 安全的查询示例
SELECT id, username, email, first_name, last_name 
FROM users 
WHERE is_active = TRUE 
ORDER BY created_at DESC 
LIMIT 10;

总结 ​

SQL是数据管理的核心技能，掌握这些基础知识将为你后续学习更高级的数据库技术打下坚实基础。记住：

  • 多练习：理论结合实践
  • 理解概念：不要死记硬背语法
  • 关注性能：学会优化查询
  • 注意安全：防范SQL注入等安全问题

继续深入学习SQL的高级特性，如存储过程、触发器、视图等，将进一步提升你的数据库技能水平。
